# Прототип автоматизации процесса закупки (To-Be)

Клиентское приложение на **Quasar (UMD/CDN)** без сервера и без сборки.  
Цель: имитация процесса "Организация и контроль поставки" из BPMN to-be.

## Что реализовано

- Каталог поставщиков.
- Каталог товаров поставщиков.
- Создание и редактирование заявок (форма с позициями).
- Обработка заявки по шагам процесса:
  - отправка на проверку,
  - автопроверка ИС (ветка корректно/на доработку),
  - регистрация,
  - формирование заказа поставщику,
  - получение счета,
  - оплата,
  - уведомление поставщика,
  - контроль срока и напоминания,
  - приемка на складе,
  - загрузка акта,
  - закрытие,
  - выдача товара заказчику.
- Ролевая модель действий: заказчик, снабжение, бухгалтерия, склад, ИС.
- История событий и визуальная карта движения заявки.
- Сохранение данных в `localStorage`.
- Моковые данные на русском языке.

## Файлы проекта

- `index.html` - интерфейс приложения.
- `app.js` - логика процесса и мок-данные.
- `styles.css` - стили.
- `.github/workflows/deploy-pages.yml` - автодеплой на GitHub Pages.

## Запуск локально

Можно просто открыть `index.html` в браузере.

Для корректной работы модальных окон и внешних CDN-ресурсов лучше запускать через простой локальный сервер:

```bash
npx serve .
```

или:

```bash
python3 -m http.server 8080
```

## Публикация в GitHub Pages

1. Отправьте проект в GitHub-репозиторий.
2. В репозитории откройте `Settings -> Pages`.
3. В `Build and deployment` выберите `Source: GitHub Actions`.
4. После `git push` в `main` или `master` workflow автоматически опубликует сайт.

## Примечания

- Это учебный **клиентский прототип**: интеграции с почтой, ЭДО и бухгалтерией имитированы.
- Для повторной демонстрации можно нажать кнопку `Сбросить демо-данные`.
