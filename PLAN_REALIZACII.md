5. Визуализация процесса (граф/список), подсказки по текущему этапу и роли.

## 4. Стратегия реализации

Приоритет: сначала закрыть все обязательные пункты отчета, затем (опционально) усиливать функционал UI.

### Трек A (обязательный, для сдачи)

1. Подготовить документацию и схемы по всем пунктам Этапов 1-3.
2. Зафиксировать соответствие SIPOC и `As-Is -> To-Be`.
3. Описать эффект внедрения и устраненные узкие места.

### Трек B (обязательный, для сдачи)

2. Включить эти данные в автопроверку заявки.
3. Добавить отдельный экран/отчет по заявкам, требующим решения.

## 7. Критерии готовности (Definition of Done)

1. По каждому пункту задания есть явное соответствие в отчете: `сделано / адаптировано / не требуется`.
2. Для всех обязательных пунктов приложены схемы и пояснения.
3. Описаны и подтверждены эффекты автоматизации относительно `As-Is`.
4. Прототип демонстрирует ключевой `To-Be` сценарий от создания заявки до закрытия.
5. Материалы готовы к публикации и проверке (включая GitHub Pages).
